==============================================================
  PLANTE v2 - How to Test Recon Agent with DVWA
==============================================================

PREREQUISITES:
- Kali VM running (see HOW_TO_RESUME.txt Step 1)
- DVWA installed on Kali (already done, Feb 2026)
- ZAP installed on Kali (already done, Feb 2026)
- PLANTE venv activated on Windows

==============================================================
STEP 1: Start Kali VM
==============================================================

Open PowerShell and run:

  "C:\Program Files\Oracle\VirtualBox\VBoxManage.exe" startvm "kali-linux-2025.4-virtualbox-amd64" --type headless

Wait ~30 seconds for it to boot.

==============================================================
STEP 2: SSH into Kali
==============================================================

  ssh kali@192.168.56.101

Password: kali

==============================================================
STEP 3: Start DVWA
==============================================================

In the Kali SSH session:

  sudo dvwa-start

Password for sudo: kali

DVWA will be available at: http://192.168.56.101:42001
Login: admin / password

FIRST TIME ONLY — set up the database:
  1. Open http://192.168.56.101:42001/setup.php in your browser
  2. Click "Create / Reset Database"
  3. It will redirect to login page — you're good to go

==============================================================
STEP 4: Start MCP + ZAP + DVWA (all in one)
==============================================================

In the Kali SSH session, just run:

  sudo bash /home/kali/start_plante.sh

Password for sudo: kali

This starts DVWA, ZAP daemon, and MCP server automatically.
After ~10 seconds it prints a status check:
  DVWA  : 200  ← good
  MCP   : 404  ← good (404 is normal, Flask has no root route)
  ZAP   : 200  ← good

Logs saved to: ~/mcp.log and ~/zap.log

==============================================================
STEP 6: Run Recon Test (Windows)
==============================================================

Open a NEW PowerShell window on Windows (keep SSH session open).

  cd C:\Users\danny\Desktop\Projects\PLANTE
  .\venv\Scripts\Activate.ps1
  python test_recon_agent.py

This runs the ReconAgent against DVWA and prints structured findings.
No execute agent, no coordinator — recon only.

Expected cost: ~$0.10-0.30 depending on how many tools the agent uses.

==============================================================
STEP 7: Shut Down When Done
==============================================================

On Kali (SSH session):

  sudo dvwa-stop
  sudo killall python3       (stops MCP)
  sudo killall java          (stops ZAP)

Exit SSH:
  exit

Stop Kali VM from PowerShell:
  "C:\Program Files\Oracle\VirtualBox\VBoxManage.exe" controlvm "kali-linux-2025.4-virtualbox-amd64" poweroff

==============================================================
TROUBLESHOOTING
==============================================================

Problem: DVWA shows "database error" on setup.php
  - Make sure MySQL is running: sudo systemctl status mysql
  - If stopped: sudo systemctl start mysql
  - Then try setup.php again

Problem: ZAP won't start
  - Check port 8080 not in use: sudo lsof -i :8080
  - If occupied: sudo kill -9 $(sudo lsof -t -i :8080) then retry
  - ZAP needs Java — check: java -version

Problem: zap-cli command not found
  - Install it: pip install zapcli
  - Or use full path: ~/.local/bin/zap-cli

Problem: MCP server unreachable
  - Check it's running: curl http://192.168.56.101:5000
  - Must start with --ip 0.0.0.0 (not just localhost)
  - If suspended: nohup python3 kali_server.py --ip 0.0.0.0 > /dev/null 2>&1 &
